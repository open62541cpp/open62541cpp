25/**
26 * Access control base class.
27 *
28 * Used to authenticate sessions and grant access rights accordingly.
29 * Custom access control can be implemented by deriving from this class and overwriting the access
30 * control callbacks.
31 *
32 * If exceptions are thrown within the access control callbacks, they are caught in the C callbacks
33 * and will return the most restrictive access rights, e.g. `0x00` in `getUserAccessLevel` or
34 * `false` in `allowAddNode`. A warning log message with the exception will be generated.
35 *
36 * The `sessionId` can originally be both `NULL` in open62541.
37 * This is the case when, for example, a MonitoredItem (the underlying Subscription) is detached
38 * from its Session but continues to run.
39 * This wrapper passes `session` by reference, so it can't be `NULL`.
40 * Instead, a `session` with an empty `sessionId` will be passed.
41 *
42 * @see UA_AccessControl
43 * @see https://www.open62541.org/doc/1.3/plugin_accesscontrol.html
44 */
