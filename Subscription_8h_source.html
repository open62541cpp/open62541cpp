47/**
48 * High-level subscription class.
49 *
50 * The API is symmetric for both Server and Client, although servers don't use the subscription
51 * mechanism of OPC UA to transport notifications of data changes and events. Instead MonitoredItems
52 * are registered locally. Notifications are then forwarded to user-defined callbacks instead of a
53 * remote client. The `subscriptionId` for servers is always `0U`.
54 *
55 * @tparam Connection Server or Client
56 * @note Not all methods are available and implemented for servers.
57 *
58 * Use the free functions in the opcua::services namespace for more advanced usage:
59 * - @ref Subscription
60 * - @ref MonitoredItem
61 */
